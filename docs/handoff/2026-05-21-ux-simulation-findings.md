# UX 模拟测试发现 — 2026-05-21

## 测试场景：小老板第一次打开 Holon

### Critical
| 问题 | 影响 | 修复建议 |
|------|------|---------|
| 首页加载 39 秒 | 用户直接关掉，认为软件坏了 | 加 splash/loading screen；fixture 预加载；考虑去掉 dev mode 的 on-demand compile |
| /members 14 秒 | 用户以为卡了 | 同上 — cold start 问题 |

### 注意
| 问题 | 影响 | 修复建议 |
|------|------|---------|
| /me 4.3 秒 | 可接受但不理想 | loading.tsx 已加，但首次 compile 无法避免 |
| 微信助手 API 冷启动失败 | 第一次调 /api/v1/staff 时 JSON parse 错误 | dev server 未完全就绪时应返回 loading 状态 |

### 小老板视角的体验流程
1. 安装 → 打开 → **等 39 秒（会离开）** ← CRITICAL
2. 看到 onboarding → 填名字 → 选角色 → OK（合理）
3. 进主页 → 聊天框在左 → **不知道干啥**（需要引导）
4. 想看微信 → 去 /members → 看到微信助手 → 点进去 → 发消息 → **等 10-20 秒**（需要进度提示）← 已修
5. 看到结果 → OK

### 最大的 drop-off 点
1. **第一次打开的等待时间** — 39 秒在 dev，production standalone 也有 5-10 秒
2. **不知道下一步做什么** — 没有 guided tour 或 first-action 提示
3. **微信连接配置复杂** — 需要 WeChat Desktop 登录 + read server 在跑

## 产品 Gap 分析（持续更新）

### 必须解决（影响第一印象）
1. **首页加载慢** — production standalone 也有 5-10s，需要 splash screen
2. **不知道干啥** — 新用户进来没有引导。需要：
   - 首次登录后弹一个"试试问微信助手：看看最近谁给我发微信了"
   - 或者 onboarding 最后一步直接演示一次微信读取
3. **微信连接太复杂** — 用户需要手动启动 read server。应该 Tauri 自动启动。

### 应该有（提升留存）
4. **语音消息转文字** — 微信语音消息显示 [语音]，应该用 Whisper/SenseVoice 转文字后再总结
5. **定时微信摘要** — 每天早上自动总结昨天的微信消息，push 到 /inbound
6. **联系人分组** — 客户 vs 同事 vs 朋友，不同组不同处理策略
7. **回复草稿** — 微信助手不仅总结，还能草拟回复。用户确认后通过 WeChat paste 发出

### 锦上添花（V2）
8. **群聊支持** — 读取群消息、@提取、群内 AI 摘要
9. **文件/PDF 提取** — 微信里收到的文件自动归档
10. **多设备同步** — 手机上也能看微信助手的分析结果
