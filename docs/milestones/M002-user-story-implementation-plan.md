# M002: User Story 实施计划

15 个 user story → 分批实施 → 每批交付后你体验验收

---

## Batch 1: 核心三件（Codex ~16h → 2天内交付）

直接把满足度从 0/15 提到 3/15 ✅

| Story | 实施内容 | Codex 时间 | 达到状态 |
|-------|---------|-----------|---------|
| **1.1 每日微信摘要** | wechat_contacts → 逐个 read → DeepSeek 分类总结 → 推到 /inbound | 4-6h | ✅ 自动摘要，紧急度分类 |
| **1.3 语音转文字** | silk→WAV + faster-whisper 转录 → 替换 [语音] 标记 | 6-8h | ✅ 语音消息变文字 |
| **2.1 搜索供应商报价** | 已有 wechat_search + LLM 总结增强（提取价格变化） | 2h | ✅ 关键词搜+价格提取 |

**你验收时测**：
- 打开 app → /inbound 顶部看到"微信日报"卡片
- 语音消息不再显示 [语音]，而是文字
- 微信助手输入"搜索报价相关消息" → 返回带价格变化的结果

---

## Batch 2: 扩展五件（Codex ~16h → Batch 1 后 2天）

满足度从 3/15 提到 8/15 🟡→✅

| Story | 实施内容 | Codex 时间 | 达到状态 |
|-------|---------|-----------|---------|
| **1.2 未回复检测** | 对比收发时间差，标记 pending 对话 | 4h | ✅ "3天没回的客户"列表 |
| **2.2 群消息摘要** | 群 wxid 读取 + LLM 摘要（排班/投诉/通知分类） | 2h | ✅ 群摘要 |
| **2.3 回复草拟** | LLM 根据消息内容草拟回复，用户确认后 copy | 4h | 🟡 草拟能用，不能自动发 |
| **3.1 消息分类** | triage skill 扩展到微信消息（询价/下单/物流） | 4h | 🟡 分类能用，不自动回复 |
| **5.1 预约提取** | LLM 从聊天提取日期+时间+事项 → 结构化输出 | 2h | 🟡 提取能用，无日历集成 |

**你验收时测**：
- 微信助手："谁超过3天没回我" → 列出未回复客户
- 微信助手："总结一下XXX群今天的消息" → 分类摘要
- 微信助手帮草拟回复 → 你 copy paste 到微信

| Story | 为什么不做 |
