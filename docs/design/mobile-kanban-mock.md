# 移动端看板设计 mock（待办 / 进行中 / 交付）

状态:**MOCK 待审**(2026-05-24)。先调研专业看板 → 内容规格 → 手机宽度 mock。方向确认后再上 :3006 实装。

## 1. 调研结论(专业看板通则)

- **一卡一事,卡片要瘦** —— 只放"能改变一个决策"的字段,其余别堆。需长滚动 = 该拆。(Atlassian / Wrike / Kanban Tool)
- **通用字段**:标题(意图陈述,不是"邮件计划"这种含糊词)、负责人、截止日(时间敏感时近乎必填)、优先级(**颜色编码**一眼可扫)。
- **Jira 卡片分层**:摘要(顶)→ 关键字段 → 明细(类型/优先级/负责人/估时);额外字段 ≤3。
- **列 = 工作流阶段**;WIP 上限防积压;**"在某列停留时长 / aging"** 是发现卡住任务的关键信号。
- **移动端**:紧凑、清晰优先;紧急项(expedite)红色置顶;swimlane 可按"谁/优先级"加上下文。

## 2. 我们的不同:这不是团队板,是"老板监工 AI 员工"

每一列对应一个**不同的老板决策**,卡片只暴露做"下一步动作"所需的信息:

| 列 | 含义 | 老板的决策 | 卡片必须回答 |
|---|---|---|---|
| **待办** | 还没派出去的活(backlog) | 派给谁 / 先放着 / 不做 | 是什么、多急、几时要、**怎么派** |
| **进行中** | 派了,员工在干 | 让它跑 / 看一眼 / 介入 | **谁在干**、跑多久了、活着吗、最新动静 |
| **交付** | 干完交回来了 | 验收 / 打回 / 归档 | 什么成果、谁交的、何时、能不能收 |

> 老板视角:"没分配下去的才是最大的" → **待办默认置顶**;交付去重(弱化)。

## 3. 每列内容规格

### 待办 To Do —— 未派出的 backlog
- ▎优先级色条(高红/中黄/低灰)+ 任务文字同色
- 任务标题(清晰意图)
- 📅 截止(逾期=红) · ⏳ 已挂 N 天(aging 推一把)
- 动作:**💬 对话小秘(派活)** · ✓ 完成 · 🗑 删除
- 列头:`待办 · N`

### 进行中 In Progress —— 已派、运行中
- 👤 员工头像 + 名字(**谁在干** = 监工第一信息)
- 任务标题
- ● 状态 pill:运行中(绿·脉动)/ 排队(灰)/ ⚠ 卡住(红)
- ⏱ 已跑 12 分 · 最新:"正在写单测…"(heartbeat 心跳)
- 动作:**查看实时**(进 agent 输出) · 去对话(介入)
- 列头:`进行中 · N`(WIP 感知)

### 交付 Delivered —— 交回的成果(弱化)
- 📄 类型 + 标题
- 👤 谁交的 · 🕐 何时("2 小时前 · 小赵")
- 1–2 行摘要
- ● 待验收(高亮·新)/ 已看
- 动作:**查看交付** → 验收 / 打回
- 列头:`交付 · N`

## 5. 与当前实现的差距(实装 checklist)
- 待办:已有优先级/日期/对话小秘 → 加 `⏳已挂N天` aging、色条。
- 进行中:当前只 `状态·时间·staff_id` → 加 头像+名、脉动状态、`已跑时长`、最新动静、查看实时/去对话 两按钮。
- 交付:当前 `状态·标题·摘要` → 加 谁交的+何时、待验收高亮、验收/打回。
