# Mobile Deltas — auto-fix queue + global pattern queue

Mobile track's cross-loop coordination queue. Mirror of `docs/deltas.md` for the desk track. Append-only by the mobile-QA loop. Consumed by mobile-DEV loop (local section) + mobile-REQ loop (global section).

## Conventions

- **Local delta** — a specific mobile defect tied to a specific surface / file / commit. Fixable by a small focused dev agent without re-planning. Format: `- [ ] M-L-<NNN> · <SHA-short> · <surface>: <one-line issue> → fix: <verb + target>`
- **Global delta** — a pattern across mobile surfaces, architectural drift, signal that current iter plan is mis-targeted, OR a need that crosses into desk territory (api-contract / core). Format: `- [ ] M-G-<NNN> · <evidence>: <pattern> → action: <re-plan / new ADR / new debt entry / desk-side delta>`
- Mark `[~ <claimer> <ISO>]` when in flight, `[x] <SHA>` when shipped (for M-L items) or `[x] <iter-plan-update SHA>` (for M-G items).

Don't delete shipped lines for 7 days — useful for the mobile-REQ loop to see what's been moving.

The mobile-dev-daemon picker reads `git show origin/main:docs/mobile-deltas.md` (NOT the local file) per L-013 — markers flipped on main always seen here. This is the mobile-track's day-1 protection against the marker-drift class of bugs that desk discovered the hard way.
