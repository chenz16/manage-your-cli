Holon Functional Architecture Diagrams

Standalone HTML/SVG diagrams for the core functional architecture: local team routing, peer mission handoff, completion callback, and local Hermes execution.

1. Functional Flow

One local assignment enters the node, then routes either to local AI execution or to a peer member connected to a remote Holon node. Both paths return normalized events and durable deliverables.

2. Peer Mission Sequence

The peer path is the signature Holon workflow: a local member identity represents a real person or team on a remote node.

3. Local Hermes Execution Sequence

The local AI path stays inside the node. Hermes executes bounded work, while Holon owns assignment status, events, and deliverables.

4. Handoff Lifecycle

A handoff is the accountable transfer of work across an owner boundary. Assignments and missions are local records; handoff is the bridge between them.
